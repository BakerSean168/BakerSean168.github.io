## 流式数据

- 数据分多次传输，无需等待完整内容即可开始处理
- 场景：实时数据、视频流、音频流等

## 方案

### 1. Server-Sent Events (SSE)

特点：  
- 单向通信（服务器推送数据，浏览器接收数据）
- 基于 HTTP 协议，自动重连
- 兼容性好

### 2. WebSocket

- 双向通信（服务器推送数据，浏览器接收数据）
- 适合高频双向数据交换

### 3. Fetch API + Streams API

特点：  
- 精细控制数据流
- 适合处理大文件下载
